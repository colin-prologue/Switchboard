## Resuming a session (handoff protocol)

This repo **is** the handoff â€” sessions are disposable; start a fresh one per merged sub-plan (the merge is a clean phase boundary that avoids context rot). No transcript is needed to pick back up.

- **To resume cold:** read in order â†’ the State block below â†’ [docs/ROADMAP.md](docs/ROADMAP.md) â†’ any `pending-review`/`proposed` records in [decisions/](decisions/) â†’ the active plan's `- [ ]` checkboxes + `git log`. The State block's **"Next:"** line is the single starting action.
- **Before `/clear` or switching sessions (the handoff):** (1) commit + push all work â€” nothing may live only in the worktree or transcript; (2) update the State block (what's done + the one "Next:" action); (3) flush loose context â€” a settled decision â†’ an ADR in `decisions/`, an open question â†’ a ROADMAP note. Optionally run `/oracle-preclear` first to capture any cross-project philosophy before clearing.
- **Litmus test:** could a brand-new session start the "Next:" action from these files alone, with zero reference to the prior conversation? If not, something is still transcript-only â€” capture it.
